# 02 — 技术方案

## 技术栈

| 层 | 选择 | 版本 | 说明 |
|---|---|---|---|
| 页面结构 | HTML5 | — | 语义化标签 + `<template>` 模板 |
| 样式 | CSS3 | — | 使用 CSS 变量、flexbox、grid |
| 逻辑 | Vanilla JS | ES6+ | 无框架，零构建工具 |
| 存储 | IndexedDB | — | 通过 Dexie.js 3.x CDN 封装 |
| 离线 | Service Worker | — | Cache-First 策略 |
| 部署 | GitHub Pages | — | 免费 HTTPS 托管 |

## 为什么选纯 HTML/CSS/JS

- 用户是编程小白，不应接触 Node.js、npm、webpack 等工具链
- 文件直接双击即可在浏览器中打开测试
- 部署只需上传文件到 GitHub，无需构建步骤
- iOS Safari 对 ES6+ 支持良好（iOS 15 = Safari 15）

## 数据模型

```javascript
Entry = {
  id: String,           // crypto.randomUUID() 生成
  title: String,         // 日记标题，允许为空
  content: String,       // 正文，纯文本
  media: Array<{         // 媒体附件
    id: String,
    type: "image" | "video",
    data: ArrayBuffer,   // 文件二进制数据
    name: String         // 原始文件名
  }>,
  createdAt: Date,       // ISO 8601 字符串
  updatedAt: Date        // ISO 8601 字符串
}
```

## 存储方案

使用 Dexie.js 简化 IndexedDB 操作：

```javascript
const db = new Dexie("DiaryDB");
db.version(1).stores({
  entries: "id, createdAt"
});
```

- id 为主键
- createdAt 建索引，用于按日期排序和筛选
- 媒体文件转 ArrayBuffer 存入同一条记录

## 图片压缩策略

- 上传时用 Canvas 重绘，限制最大宽度 800px
- 保持宽高比，JPEG 质量 0.7
- 避免 IndexedDB 存储空间过快耗尽

## Service Worker 策略

- 预缓存：index.html, css/style.css, js/*.js, assets/icon.svg
- 运行时缓存：CDN 引入的 Dexie.js
- 策略：Cache-First（优先从缓存读取，离线可用）
